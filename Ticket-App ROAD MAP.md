# **Ticket-App ROAD MAP**

1. Initialize
   - Stack: EJS, Express, JS, HTML, CSS, Bootstrap, MongoDB
   - A few low fidelity prototyped sections
   - GitHub initialization
   - Defining brief scope of project
2. Ticket Overview Section
   - Defining standard view of ticket list
   - Create better UI
3. Creating User
   - Admin can create any user
   - Managers can create basic users
     - Separate manager only page that only displays if logged in user type is manager, writes new basic user to db
   - Basic users cannot create any users
4. Left Side Nav-Bar
   - Navbar on left side of screen
   - My Tickets
     - Overview for tickets assigned to current user
     - Settings related to UI design
     - User profile holds email, statistics, etc
5. Detailed Ticket information
   - More in depth details related to ticket
   - Text area that the ticket holds
   - Message thread
   - File upload
   - Notifications
6. Creating Tickets
   - Allow users to create tickets and add ticket to DB
